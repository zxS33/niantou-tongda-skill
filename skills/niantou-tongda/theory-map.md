# 念头通达理论地图

本文件用于 README、文案、方法论解释和扩展设计。普通执行任务只需读 `SKILL.md`。

## 一、道家：清镜、无滞、顺势

- 《道德经》“涤除玄览”：清理内在镜面，让判断不被欲望和恐惧污染。
- 《庄子》“心斋”：让心不被耳目成见牵走，回到虚静与现实。
- 《庄子》“坐忘”：把身份、执著、成败感放下，不把一次结果当成自我。
- 工程映射：先清事实，再建假说；不把代码、方案、设计稿当成自我。

## 二、儒家：反求诸己，义利分明

- 反求诸己：把能动性收回自己可改进处。
- 义利之辨：做决定时问“守住什么”，而不是只问“赚到什么”。
- 日参省乎己：复盘应成为常态机制。
- 工程映射：复盘不自责不甩锅，取舍要有价值标准。

## 三、禅宗：无住行动

- “应无所住而生其心”：不执著于结果、评价、身份和旧方案。
- 工程映射：先交最小可反馈版本，把反馈当信息，不当审判。

## 四、斯多葛：可控边界

- Epictetus 的核心区分：有些事在我们掌控内，有些不在。
- 工程映射：外部 review、排期、市场反馈不可控，但说明、测试、替代方案、沟通节奏可控。

## 五、现代心理学：认知拆解与价值行动

- CBT：识别灾难化、读心术、全或无、人格化等自动想法。
- ACT：认知解离、价值澄清、承诺行动。
- 工程映射：把“我不行”改成“这个假说未验证”；把“怕被批”改成“小 PR 获取反馈”。

## 六、工程方法：探针、闭环、复利

- OODA：观察、定位、决策、行动循环。
- 敏捷：工作软件、客户协作、响应变化。
- Blameless postmortem：关注系统改进，不把复盘变成追责。
- 工程映射：每个瓶颈都要设计最小探针；每次失败都换回一条新信息。

## 七、修仙叙事：韩立式道心

本项目借用《凡人修仙传》相关讨论中的“韩立式念头通达”作为文化入口，但只做方法论转译：

- 小绿瓶：工具和资源可以放大能力，但不能替代判断。
- 灵根：初始条件有限时，靠长期复利、谨慎试错和机会管理破局。
- 化凡入世：回到真实世界，看见用户、日志、现场和反馈。
- 藏锋守拙：证据不足时不逞强，条件成熟时小而准地出手。
- 飞升回望：最终收念标准不是一时爽，而是回头看这一步是否值得。

工程映射：agent 要先承认约束，再识别工具和后手；用最小探针换事实，用小 diff 出手，用复盘收念。

## 总结

念头通达不是情绪管理单点技巧，而是一套 agent 工作法：

```text
明镜看事实 -> 诊断心结 -> 划清边界 -> 本心决断 -> 无住行动 -> 破境探针 -> 工程交付 -> 长线守势 -> 收念闭环
```
