# 长线检查表

行动前检查：

- [ ] 这一步是否关闭关键退路？
- [ ] 证据是否足以支撑行动强度？
- [ ] 是否为了证明自己而不是服务目标？
- [ ] 是否增加长期维护债？
- [ ] 三个月后我是否仍认可这个选择？
- [ ] 有没有更小、更稳、更可逆的动作？
- [ ] 我是在求真实通达，还是只想一时痛快？
- [ ] 我有没有识别可复用工具，而不是迷信工具？
- [ ] 我是否承认当前约束，并为约束设计后手？

## 重手动作降级表

| 冲动动作 | 长线替代 |
|---|---|
| 重写整个模块 | 先加测试封住行为 |
| 激烈反驳 reviewer | 列事实和 tradeoff |
| 接下所有需求 | 给范围和排期选项 |
| 公开指责 | 先收集证据和私下沟通 |
| 推翻设计 | 做对照稿验证问题 |
| 迷信模型一次生成 | 加验收标准、样例和人工复核 |
| 为了快而省测试 | 降低范围，但保留关键验证 |
| 因为被否定而沉默 | 先补证据，再用低冲突方式同步 |
