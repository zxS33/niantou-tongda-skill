# 通达工作流——理论依据

## 方法组合

单个方法解决一个认知动作；工作流解决动作顺序。顺序错了，方法越多越乱。

## 工程流程映射

- 调试：观察 -> 假说 -> 探针 -> 决策 -> 复盘。
- 产品：反馈 -> 归因 -> 原型 -> 验证 -> 迭代。
- 项目管理：边界 -> 取舍 -> 同步 -> 执行 -> 闭环。

## 念头通达映射

心法不是停在心里，而是落到任务状态转移：

```text
滞碍 -> 分层 -> 诊断 -> 选择 -> 行动 -> 验证 -> 收念
```
