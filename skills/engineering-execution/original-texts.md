# 思想来源摘录

这些来源只作为方法论锚点，不要求 agent 在交付中引用。

## 尊重事实的工程化

先看事实，再做判断。对 agent 来说，事实来自代码、日志、截图、测试、数据、用户原话和可复现行为。

## 庖丁解牛

遇到复杂系统，不靠蛮力硬切，而是识别结构、关节和阻力最小处。工程里对应最小探针、单变量验证和局部改动。

## OODA

观察、判断、决策、行动形成循环。重要的是循环速度和反馈质量，而不是一次性写出完美方案。

## Blameless postmortem

复盘不做人格审判，只保留事实、责任、系统改进和后续动作。它与“收念闭环”同向：带走规律，停止重复消耗。

## 专业交付

专业不是显得很懂，而是让别人能使用结果、复查证据、理解风险，并在需要时继续推进。
