# 认知陷阱速查表

| 陷阱 | 内心台词 | 工程表现 | 明镜修正 |
|---|---|---|---|
| 灾难化 | “这下完了” | 一个测试失败就认为架构全错 | 查影响范围 |
| 读心术 | “他就是看不起我” | 把评审意见当人格攻击 | 回到文字事实 |
| 全或无 | “不完美就没用” | 不交原型、不发小 PR | 设足够好标准 |
| 人格化 | “都是我的问题” | 承担系统和流程缺陷 | 分责任边界 |
| 确认偏误 | “我早就知道” | 只找支持自己方案的证据 | 主动找反例 |
| 沉没成本 | “都写这么多了” | 不愿删错方向代码 | 看未来价值 |
| 面子执著 | “我不能被否定” | 评审中防御性解释 | 改成问题求证 |
